04

Strategy

  1. S0 first, alone. Every red-now case reads the counters; without them the RT claims are vibes.
  2. Then per step: red → fix → green, one case at a time (vertical slices). Never batch reds.
  3. Genuinely-new pins before the refactor they guard (T1.3–1.5, T2.5–2.6, T5.1, T6.1, T7.1, T8.3, T9.1–9.2); existing suites are the regression net and are never re-authored.
  4. No shared oracles: facts vs hand-seeded expectations, golden vectors frozen as literals, hand-coded payloads — an equivalence test that calls the code-under-test's own helper proves nothing.
  5. Batch statements never throw — absent rows return empty; fault-injection cases (T1.7, T6.1, T9.3) pin status precedence.
  6. End of each phase: thermo-nuclear gate → fix findings → full suite green → commit before the next phase.
